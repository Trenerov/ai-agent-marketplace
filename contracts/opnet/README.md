# OP_NET Contracts Workspace

This workspace contains the smart-contract layer for the AI Agent Marketplace described in the technical specification.

Contracts included:

- `AgentNFT` - mintable OP_721 agent ownership contract
- `AgentRegistry` - discovery and ranking layer
- `UsagePayment` - pay-per-use and revenue split engine
- `Marketplace` - fixed-price secondary market

Notes:

- The code is structured against the official `@btc-vision/btc-runtime` AssemblyScript runtime.
- Runtime APIs evolve quickly. Verify decorator names, ABI types, and storage helpers against the installed runtime version before deployment.
- The Next.js app does not depend on this folder at build time.

Recommended setup:

```bash
cd contracts/opnet
npm install
npm run build
npm test
npm run deploy:plan
npm run deploy:dry-run
```

Deployment files:

- `deployment/manifest.json` contract order and configure dependencies
- `deployment/config.example.json` RPC, fee and wallet defaults
- `deployment/state.example.json` expected persisted deployment shape
- `deployment/plan.json` generated by `npm run deploy:plan`

Deployment scripts:

- `npm run deploy:plan` builds a machine-readable plan from manifest plus known addresses
- `npm run deploy:dry-run` prints the current deployment/configure sequence without broadcasting
- `npm run deploy` deploys missing contracts and runs configure calls using `OPNET_MNEMONIC`
- `npm run deploy:sync-env` writes app-level `.env.contracts.local` from deployment state
- `npm run broadcast:intent` executes a contract intent payload from stdin using the local OP_NET SDK
- `npm run broadcast:signed` broadcasts already-signed raw transaction payloads from stdin
- `npm run prepare:intent` simulates a contract intent payload from stdin and emits `toOfflineBuffer()` bundles for offline signing
